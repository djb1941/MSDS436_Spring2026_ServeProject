PHASE 1: DEPENDENCY FREEZING
================================================================================
Status: [ ] TODO

Before committing final code, freeze all Python package versions:

Steps:
  1. Set up local Python environment:
     python3 -m venv venv
     source venv/bin/activate

  2. Install all requirements:
     pip install -r requirements-python.txt
     pip install -r requirements-webapi.txt

  3. Test your application locally to ensure everything works

  4. Capture exact versions installed:
     pip freeze > requirements-python-frozen.txt
     pip freeze > requirements-webapi-frozen.txt

  5. Update the actual requirements files with frozen versions:
     cp requirements-python-frozen.txt requirements-python.txt
     cp requirements-webapi-frozen.txt requirements-webapi.txt

  6. Commit frozen versions to version control

Why: Docker relies on these exact versions for reproducibility. Your professor
needs to be able to run your code and get identical results.

NOTES & DECISIONS LOG
================================================================================

Technology Choices Made:
- Base OS: Fedora 40 (aligns with RHEL career focus)
- Package Manager: dnf (instead of apt-get)
- Database: PostgreSQL with PostGIS (geospatial queries)
- Python Framework: FastAPI (modern, async, easy to document)
- Python Version: 3.11 (stable, good library support)
- No pinned versions initially (flexibility, pip resolves at build time)

Known Limitations/TODOs:
- [ ] PostgreSQL still uses Debian-based image (official image doesn't have Fedora variant)
- [ ] MobilityDB extension commented out (may need additional setup)
- [ ] Geospatial visualization could be enhanced with real-time updates

Design Decisions:
- Separate services for database, simulation, API (separation of concerns)
- Docker network isolation (database not exposed to outside)
- Health checks to ensure service readiness
- Volume mounts for development (hot reload)
